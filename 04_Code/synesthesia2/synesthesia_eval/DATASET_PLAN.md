# 📊 תוכנית איסוף ותיוג Dataset - Synesthesia Evaluation

## 🎯 מטרה
לבנות dataset של סרטוני audio-visualization עם ציוני ground truth לאימון וולידציה של מודל הערכת איכות.

---

## 📈 גודל Dataset מומלץ

| Split | כמות מינימלית | כמות אידיאלית | סיבה |
|-------|---------------|---------------|------|
| Train | 50 | 100-150 | מספיק ל-Ridge regression עם ~10 features |
| Validation | 15 | 30 | כיול ו-early stopping |
| Test | 15 | 30 | הערכה סופית לא-מוטה |
| **סה"כ** | **80** | **160-210** | |

> 💡 עבור k-fold CV, אפילו 50-80 דוגמאות יכולות לתת תוצאות משמעותיות

---

## 📹 מקורות סרטונים

### 1. סרטונים קיימים (הכי מהיר!)
| מקור | כמות משוערת | יתרון | חיסרון |
|------|-------------|-------|--------|
| ערוץ YouTube שלך | ~50-100 | כבר קיימים, מגוון סגנונות | צריך להוריד |
| סרטוני פיתוח (תיקייה) | ~20-30 | זמינים מיידית | רק Synesthesia |
| פרויקטים ישנים | ~10-20 | מגוון | צריך לחפש |

### 2. יצירה אוטומטית (variations)
- להריץ את ה-renderer על אודיו עם פרמטרים שונים:
  - `fade_rate`: [0.85, 0.90, 0.95]
  - `accumulation_strength`: [0.3, 0.5, 0.7]
  - `temporal_blend`: [0.2, 0.4, 0.6]
- **פוטנציאל:** 20-30 variations מכל שיר

### 3. "מתחרים" (למגוון)
- חיפוש YouTube: "music visualization", "audio reactive"
- שימוש הוגן למחקר
- **פוטנציאל:** 20-30 דוגמאות

---

## 🏷️ אסטרטגיית תיוג

### Option A: תיוג עצמי (מומלץ להתחלה)
**יתרונות:**
- מהיר, בשליטה מלאה
- לא צריך לשלם/לארגן אנשים

**חסרונות:**
- bias אפשרי
- אין inter-rater reliability

**זמן משוער:** 2-3 דקות לסרטון × 80 = ~4 שעות

### Option B: מספר מתייגים (לשלב מאוחר יותר)
- 3-5 אנשים (חברים, משפחה, Fiverr)
- כל סרטון מקבל 3 ציונים → ממוצע
- מאפשר חישוב ICC, Krippendorff α

### Option C: Crowdsourcing (לסקייל גדול)
- MTurk, Prolific
- יקר יותר, אבל scalable
- שמור לגרסה 2.0

## 🛠️ כלי תיוג

### אופציה 1: CLI פשוט (הכי מהיר לבנות)
```bash
python label_video.py --video path/to/video.mp4

# מציג את הסרטון, שואל 3 שאלות:
# Sync (0-10): _
# Alignment (0-10): _  
# Aesthetic (0-10): _
# → שומר ל-annotations.json
```

### אופציה 2: Web UI (יותר נוח)
- Streamlit / Gradio
- Player + 3 sliders
- Preview + submit
- **זמן פיתוח:** 2-3 שעות

### אופציה 3: Spreadsheet (zero code)
- Google Sheets עם לינקים לסרטונים
- עמודות: video_id, sync, alignment, aesthetic
- **זמן פיתוח:** 30 דקות

## 📋 Workflow מוצע

### שלב 1: הכנה (1-2 שעות)
1. ✅ להוריד 50 סרטונים מ-YouTube שלך
2. ✅ לאסוף סרטוני פיתוח מהתיקייה
3. ✅ לחתוך לקליפים של 15-30 שניות (FFmpeg)
4. ✅ לארגן בתיקייה `data/raw/`

### שלב 2: כלי תיוג (1-2 שעות)
1. ✅ לבנות CLI labeler פשוט
2. ✅ לבדוק על 5 סרטונים

### שלב 3: תיוג ראשוני (3-4 שעות)
1. ✅ לתייג 50-80 סרטונים (Niv)
2. ✅ לשמור ב-JSON לפי הסכמה

### שלב 4: אימון ראשוני
1. ✅ להריץ `ScoringModel.fit(dataset)`
2. ✅ לבדוק feature importance
3. ✅ להריץ cross-validation

### שלב 5: הרחבה (אופציונלי)
1. להוסיף מתייגים נוספים
2. לחשב inter-rater agreement
3. להגדיל ל-150+ דוגמאות
